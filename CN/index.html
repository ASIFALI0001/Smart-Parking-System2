üì° Protocols Used

Communication standards powering our network

HTTP
HTTP/HTTPS

Click to learn more

HTTP/HTTPS Protocol

Used for RESTful API communication between server and client applications.

  • Port: 443 (HTTPS)
  • Layer: Application
  • Security: TLS 1.3
  • Methods: GET, POST, PUT, DELETE

In SmartPark, HTTPS secures all API calls between sensors, server, and dashboard.

MQTT
MQTT

Click to learn more

MQTT Protocol

Lightweight messaging protocol for sensor data transmission.

  • Port: 1883/8883 (TLS)
  • Layer: Application
  • Pattern: Publish/Subscribe
  • QoS Levels: 0, 1, 2

SmartPark uses MQTT for real-time sensor updates with minimal bandwidth.

TCP/IP
TCP/IP

Click to learn more

TCP/IP Protocol Suite

Foundation protocol suite for reliable data transmission.

  • Layer: Transport/Network
  • Connection: Connection-oriented
  • Features: Flow control, error recovery
  • Addressing: IPv4/IPv6

TCP ensures reliable data delivery between all SmartPark components.

802.11
Wi-Fi (802.11)

Click to learn more

Wi-Fi (802.11) Protocol

Wireless networking standard for device connectivity.

  • Frequency: 2.4/5GHz
  • Layer: Data Link/Physical
  • Security: WPA2-Enterprise
  • Standards: 802.11n/ac/ax

SmartPark sensors connect wirelessly using Wi-Fi for flexibility and ease of deployment.
